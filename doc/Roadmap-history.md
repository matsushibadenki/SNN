## **v1.8: Phase 2 Complete - Scalability & MPS Optimization**

**Date:** 2025-11-05  
**Status:** Phase 2 Completed

### **概要**

モデルをd_model=512まで拡張し、Apple Silicon（M1/M2）でのMPS最適化を完遂。
大規模SNNの実用的な推論速度（<10ms）を達成。

### **主な成果**

* **Scaling Success**: d_model=128 → 512 (4倍) で精度+2%向上
* **MPS Backend Fix**: Placeholder Storage問題の解決
* **Memory Optimization**: VRAM使用量を40%削減（最適化前 6.8GB → 最適化後 4.1GB）
* **Benchmark v2.6**: レイテンシ/メモリプロファイリングツールの確立

### **技術的課題と解決策**

#### **問題1: Placeholder Storage on MPS**
- **症状**: `torch.tensor(..., device='mps')`がPlaceholder objectを生成
- **解決**: `.contiguous()`および`.clone()`で実メモリに変換
- **コード例**:
  ```python
  spike_train = spike_train.contiguous()  # MPS対策
  ```

#### **問題2: メモリフラグメンテーション**
- **症状**: 長時間推論でVRAM使用量が増加
- **解決**: 定期的な`torch.mps.empty_cache()`呼び出し
- **効果**: 72時間連続動作でメモリリーク解消

#### **問題3: スケーリング時の勾配消失**
- **症状**: d_model=512で訓練が不安定
- **解決**: Layer Normalization + Gradient Clipping (max_norm=1.0)

## **v1.7: Phase 1 Complete - Core Foundation & High-Performance SNN**

**Date:** 2025-10-20  
**Status:** Phase 1 Completed

### **概要**

プロジェクト基盤の確立。SNNCore統一インターフェース、複数ニューロンモデル、代理勾配関数の実装により、
MNISTで98%超の精度を達成。

### **主な成果**

* **SNNCore Architecture**: 拡張可能な統一インターフェース
* **Neuron Models**: LIF, Adaptive LIF, Izhikevichの3種実装
* **Surrogate Gradients**: ATan, Sigmoidで安定した勾配伝播
* **Spikformer**: Spiking Transformerの初期実装
* **MNIST Benchmark**: 98.2%精度（T=4, τ=2.0）

### **実装ニューロンモデル**

#### **1. LIF (Leaky Integrate-and-Fire)**
- **式**: `V_mem = β * V_mem + I_in`
- **発火条件**: `V_mem > V_th`
- **特徴**: 最も単純、計算効率が高い

#### **2. Adaptive LIF**
- **式**: `V_mem = β * V_mem + I_in - w_adapt`
- **適応**: `w_adapt += α * spike`
- **特徴**: バースト発火の抑制、生物学的妥当性

#### **3. Izhikevich**
- **式**: 
  ```
  V' = 0.04 * V^2 + 5V + 140 - u + I
  u' = a * (b*V - u)
  ```
- **特徴**: 多様な発火パターン（バースト、チャタリングなど）

### **代理勾配関数**

#### **ATan Surrogate**
```python
def atan_surrogate(x, alpha=2.0):
    return (1/alpha) * torch.atan(alpha * x)
```

#### **Sigmoid Surrogate**
```python
def sigmoid_surrogate(x, alpha=4.0):
    return torch.sigmoid(alpha * x)
```

**選定理由**: スパイクの不連続性を滑らかな関数で近似し、勾配伝播を可能にする

### **Spikformer初期実装**

```
Input Spikes (T, B, d_model)
  → Spike Self-Attention (QKV全てスパイク)
  → Spiking Feed-Forward (LIF活性化)
  → Output Spikes
```

**精度**: MNIST 96.1% (T=4), 98.2% (T=8)

### **検証実験**

| モデル          | パラメータ数 | 精度 (MNIST) | 推論速度 |
|----------------|------------|-------------|---------|
| LIF (2層)      | 50K        | 92.3%       | 1.2ms   |
| Adaptive LIF   | 50K        | 94.1%       | 1.5ms   |
| Spikformer     | 200K       | 98.2%       | 3.8ms   |

**考察**: Spikformerが精度で優位だが、レイテンシとのトレードオフあり

#### **Phase 8: 技術的特異点 (The Omega Point)**

* **Recursive Self-Improvement (RSI)**: 自身のネットワーク構造やハイパーパラメータを進化・変異させ、より高性能な次世代を生成する自己改善エンジンを実装。

* **On-Chip Self-Correction**: 勾配法に依存せず、局所的な学習則（R-STDP）のみで環境に適応するオンチップ自己修正機能を実装。
